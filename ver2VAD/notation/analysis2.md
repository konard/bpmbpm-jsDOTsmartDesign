# Зачем существует много VAD-подобных нотаций

VAD-подобные нотации появились из разных практических задач. ARIS VAD описывает цепочку создания ценности и окружение функций. BCD описывает способности бизнеса как устойчивую архитектурную структуру. SILA Union VAD адаптирует VAD к регламентному описанию процессов. Stormbpmn BCD использует BPMN-палитру, чтобы не вводить новый инструмент. ArchiMate описывает те же элементы как часть корпоративной архитектуры.

Разнообразие сохраняется потому, что каждая нотация оптимизирована под свой контекст: методологию ARIS, карту способностей, процессный регламент, BPMN-инструмент или архитектурную модель. Визуально эти схемы похожи, но вопрос "что означает главный прямоугольник" отличается: функция, способность, операция, задача или бизнес-процесс.

Вариант унификации - использовать общую промежуточную метамодель:

| Унифицированный тип | Назначение |
| --- | --- |
| Work unit | Функция, способность, операция, задача или процесс |
| Event | Событие или технический маркер состояния |
| Input artifact | Ресурс, входной документ, данные |
| Output artifact | Продукт, результат, выходной документ |
| Actor | Роль, владелец, подразделение, дорожка |
| Application | ИС или прикладной компонент |
| Comment | Комментарий или annotation |

Такой слой позволяет хранить одну таблицу и затем строить разные визуализации. Для строгой семантики экспорт должен указывать целевую нотацию: например, `Work unit` становится `Business capability` в BCD, `Function` в ARIS и `Business process` в ArchiMate.
