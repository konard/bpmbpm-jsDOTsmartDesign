# VAD-образные нотации

Авторы статьи про BCD используют название "диаграмма бизнес-способностей", потому что такая схема описывает не столько ход работ во времени, сколько устойчивые способности организации: что бизнес должен уметь делать, какие результаты и ресурсы нужны, кто отвечает и какие зависимости есть между способностями. Для верхнеуровневого анализа это снижает привязку к текущему регламенту процесса и делает модель ближе к архитектурному описанию.

## ARIS VAD

Детальное описание: https://docs.aris.com/10.0.27.0/yay-method-reference/en/#/home/494393/en/1

| Элемент | Семантика |
| --- | --- |
| Function | Работа или процесс верхнего уровня, создающий ценность |
| Event | В классической VAD обычно не является основным элементом; в приложении отображается технической точкой |
| Information object | Входной или выходной документ/данные |
| Organizational unit / Role | Исполнитель или владелец работы |
| Application system | Поддерживающая ИС |
| Precedence / hasNext | Последовательность работ |
| Input / Output | Потребление и создание информационного объекта |
| Assignment | Связь работы с ролью или системой |

## BCD VAD

Детальное описание: https://www.osp.ru/os/2026/02/13060742

| Элемент | Семантика |
| --- | --- |
| Business capability | Бизнес-способность, устойчивая возможность организации |
| Event | Не является ключевым элементом; в приложении отображается технической точкой |
| Input / output artifact | Информационный вход или результат способности |
| Owner / performer | Ответственная роль |
| Dependency | Зависимость между способностями |
| Responsibility | Ответственность роли за способность |

## SILA Union VAD

Детальное описание: https://bpm3.ru/articles/modelirovanie-protsessov-v-notatsii-vad-v-sila-union/

| Элемент | Семантика |
| --- | --- |
| Process / operation | Работа верхнего уровня |
| Document | Входной или выходной документ |
| Role | Исполнитель |
| Information system | Система поддержки |
| Flow | Порядок выполнения работ |
| Document relation | Использование или получение документа |
| Performer relation | Назначение исполнителя |

## Stormbpmn BCD через BPMN

Детальное описание: https://helpdesk.stormbpmn.com/hc/docs/articles/1749580250-

## ArchiMate VAD-like

Детальное описание: https://pubs.opengroup.org/architecture/archimate3-doc/

| Элемент | Семантика |
| --- | --- |
| Business event | Событие, запускающее или завершающее поведение |
| Business process | Верхнеуровневая работа/цепочка ценности |
| Business role | Ответственная роль |
| Business object | Данные или документ |
| Application component | Поддерживающая ИС |
| Triggering | Поток события/поведения |
| Access | Доступ процесса к данным |
| Assignment / serving | Назначение роли и сервис системы |

## Различия, плюсы и минусы

| Нотация | Семантический фокус | Плюсы | Минусы |
| --- | --- | --- | --- |
| ARIS VAD | Поток верхнеуровневых функций и окружение функций | Хорошо подходит для описания цепочки создания ценности и связки с ARIS-методологией | События вторичны, детальная логика процесса выносится в EPC/BPMN |
| BCD VAD | Карта бизнес-способностей | Лучше отделяет целевые возможности бизнеса от текущего процесса | Может быть менее очевидна как исполняемый процесс |
| SILA Union VAD | Практическая процессная VAD с ролями, документами и системами | Удобна для регламентного описания верхнего уровня | Зависит от конкретной реализации инструмента |
| Stormbpmn BCD | BCD через знакомую BPMN-палитру | Легко использовать в BPMN-инструменте, есть события | BPMN-символы могут смешивать смысл способности и процесса |
| ArchiMate VAD-like | Архитектурная модель поведения, ролей, данных и приложений | Богатая семантика отношений, есть событие | Требует знания ArchiMate и менее похожа на классическую VAD |
