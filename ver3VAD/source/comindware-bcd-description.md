# Comindware BCD - описание источника

**Источник:** база знаний Comindware Platform - <https://kb.comindware.ru/> (раздел "Моделирование. Бизнес-способности").

> Примечание: полный текст статей kb.comindware.ru защищён авторским правом и не дублируется в репозитории. Ниже изложение собственными словами и табличные конспекты.

## Краткое содержание (своими словами)

Comindware Platform предлагает BCD в качестве одного из методов моделирования верхнего уровня. В отличие от OSP BCD, Comindware акцентирует внимание не на ресурсах, а на иерархической декомпозиции способностей по уровням L1/L2/L3. Ресурсы и продукты в палитре отсутствуют как самостоятельные узлы - они описываются текстовым атрибутом способности.

Ключевые отличия от OSP BCD:

1. **Иерархия важнее ресурсов.** Главная связь - "декомпозиция" (parent → child), а не "потребляет/производит".
2. **Ответственный встроен в блок.** В Comindware ответственный (Owner) - это атрибут способности, в DOT мы добавляем его к подписи блока через `\n` ("Ответственный: ...").
3. **События не моделируются.** Если в таблице есть `type=event`, мы выводим его серой "заметкой вне BCD".
4. **Цветовая палитра кодирует уровень.** На реальных диаграммах Comindware уровни L1/L2/L3 окрашены разной интенсивностью одного оттенка - в DOT мы циклически выбираем из четырёх индиго-оттенков.

## Таблица элементов нотации Comindware BCD

| Элемент              | Графика                               | Назначение                                 |
| -------------------- | ------------------------------------- | ------------------------------------------ |
| Capability           | Скруглённый прямоугольник, заливка по уровню | Способность (узел диаграммы)          |
| Owner (атрибут)      | Подпись внутри блока                  | Кто отвечает за способность                |
| Comment (атрибут)    | Подпись внутри блока                  | Произвольное пояснение                     |

## Таблица связей Comindware BCD

| Связь            | Источник     | Цель           | Стиль           |
| ---------------- | ------------ | -------------- | --------------- |
| decomposition    | Capability   | Capability     | Сплошная стрелка |

В отличие от OSP BCD здесь только одна связь - декомпозиция. Это упрощение делает Comindware BCD ближе к "карте компетенций", чем к ресурсной модели.

## Соответствие колонкам нашей таблицы

| Колонка `ver3VAD` | Comindware BCD               |
| ----------------- | ---------------------------- |
| `type=activity`   | Capability                   |
| `type=event`      | Не моделируется (помечается как "вне BCD") |
| `name`            | Имя способности              |
| `input`           | Не используется              |
| `output`          | Не используется              |
| `role`            | Owner (атрибут)              |
| `system`          | Не используется              |
| `comment`         | Comment (атрибут)            |

## Критическое замечание

В Comindware BCD легко "выродиться" в обычное дерево, а не в карту. Если все способности линейно соединены через decomposition, это уже не BCD, а упрощённая иерархическая модель оргструктуры. Реальная BCD в Comindware - это многоуровневая mind-map-подобная карта, чего в нашем линейном табличном вводе не воспроизвести (для этого нужна вложенная структура). Это сознательное упрощение, отражённое в комментарии к JS-модулю.
