# OSP BCD - описание источника

**Источник статьи:** Открытые системы. СУБД, 2026, № 2: <https://www.osp.ru/os/2026/02/13060742>

> Примечание: полный текст статьи защищён авторским правом издательства "Открытые системы" и не дублируется в репозитории. Ниже изложение собственными словами и табличные конспекты, необходимые для построения нотации `osp-bcd` в нашем коде.

## Краткое содержание (своими словами)

Статья посвящена применению Business Capability Diagram (BCD) в практике корпоративной архитектуры. Главная мысль автора: BCD - это не процессная модель, а статическая карта устойчивых способностей предприятия. На диаграмме не должно быть стрелок "что после чего"; вместо этого показывают, какая способность какой ресурс потребляет и какой продукт создаёт.

Ключевые тезисы:

1. **Способность ≠ процесс.** Если на схеме появилась стрелка "следующий шаг", это уже не BCD, а VAD или BPMN.
2. **Группировка способностей.** Способности объединяются в домены (группы) - например, "Продажи", "Поддержка". Группа - не самостоятельный узел верхнего уровня, а контейнер.
3. **Ресурсы/продукты единой природы.** Один и тот же артефакт может быть продуктом одной способности и ресурсом для другой - поэтому в нотации `osp:Resource` это один класс, а не два.
4. **Ответственный - атрибут, а не отдельный узел.** В отличие от ARIS VAD, BCD не выводит роль как самостоятельный объект.

## Таблица 1 - Элементы нотации OSP BCD (конспект)

| Элемент            | Графика                             | Назначение                                   |
| ------------------ | ----------------------------------- | -------------------------------------------- |
| Capability         | Сплошной прямоугольник              | Устойчивая способность предприятия           |
| Capability group   | Прямоугольник с пунктирной границей | Домен/группа способностей                    |
| Resource / Product | Папка (folder)                      | Ресурс на входе или продукт на выходе        |
| Owner              | Не выводится как узел               | Указывается атрибутом внутри блока (опционально) |
| Comment            | Стикер-заметка                      | Пояснение (расширение, не из ядра нотации)   |

## Таблица 2 - Допустимые связи OSP BCD (конспект)

| Связь        | Источник           | Цель               | Стиль линии        |
| ------------ | ------------------ | ------------------ | ------------------ |
| consumes     | Capability         | Resource           | Пунктирная стрелка |
| produces     | Capability         | Resource (Product) | Пунктирная стрелка |
| is-part-of   | Capability         | Capability group   | Сплошная линия     |
| has-comment  | Любой узел         | Comment            | Пунктирная (доп.)  |

**Принципиально отсутствует:** связь "следующая способность" (hasNext / sequence flow / triggering). Её отсутствие - формальный признак, что перед нами действительно BCD, а не VAD.

## Соответствие колонкам нашей таблицы

| Колонка `ver3VAD`       | OSP BCD                                |
| ----------------------- | -------------------------------------- |
| `type=activity`         | Capability                             |
| `type=event`            | Не моделируется (рендерится как plaintext) |
| `name`                  | Подпись способности                    |
| `input`                 | Resource (consumes)                    |
| `output`                | Resource/Product (produces)            |
| `role`                  | Capability group (is-part-of)          |
| `system`                | Не используется                        |
| `comment`               | Comment (опционально)                  |
